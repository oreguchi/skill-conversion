# Approved-Additions Register

Instantiate this table at Phase 0 §5 and keep updated through Phases 1 and 4.

| Added-on | Location | Summary | Reason | Source-phase | Impact | Approver |
|---|---|---|---|---|---|---|

## Column definitions

- **Added-on:** absolute YYYY-MM-DD
- **Location:** file + section anchor, e.g., `SKILL.md §3.2` or `references/patterns.md §Binding`
- **Summary:** one-line gist
- **Reason:** why this addition is justified despite being absent in source
- **Source-phase:** `Phase 1` or `Phase 4`
- **Impact:** `minor` / `moderate` / `heavy` — degree of drift from source intent
- **Approver:** user identifier

No empty Reason or Impact cells — mandatory on every row.
